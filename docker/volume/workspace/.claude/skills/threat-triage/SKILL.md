# Threat Triage Skill

## 什么时候使用

用户提供安全告警、EDR 日志、SIEM 事件、IOC、主机行为、进程链、网络连接或工单内容，需要判断风险、提取证据、输出处置建议时使用。

## 工作流程

1. 识别输入数据类型：告警、日志、IOC、工单、样本分析结果、威胁情报。
2. 提取关键实体：IP、域名、URL、Hash、文件路径、进程、命令行、用户、主机、时间。
3. 判断攻击阶段：初始访问、执行、持久化、提权、防御规避、凭证访问、横向移动、命令控制、数据外传等。
4. 组织证据链：实体、行为、时间顺序、父子进程、网络连接、命中文件或规则。
5. 给出风险等级：高 / 中 / 低，并说明置信度。
6. 给出处置建议：验证动作、隔离建议、封禁建议、溯源建议、加固建议。

## 禁止事项

- 不得直接执行封禁 IP。
- 不得直接隔离主机。
- 不得直接删除文件。
- 不得直接修改安全设备策略。
- 不得把不确定结论写成确定事实。
