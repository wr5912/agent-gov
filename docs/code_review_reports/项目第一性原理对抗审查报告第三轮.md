## 审查判据

本轮不以“测试存在”“页面有按钮”或“治理脚本通过”作为能力成立依据，而使用五条判据：

1. 行为事实只有一个权威来源，投影不能演变成并行真相源。
2. 业务动作先产生对应产物，状态推进只能是成功动作的副作用。
3. 所有权、生命周期和跨 Agent 边界必须由后端权威校验。
4. 对用户宣称完成前，失败状态、补偿、幂等和部分提交必须可证明。
5. 文档中的自动验收必须能被真实收集，并且业务语义与测试断言等价。
