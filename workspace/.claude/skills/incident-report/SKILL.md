# Incident Report Skill

## 报告结构

1. 事件摘要
2. 影响范围
3. 时间线
4. 证据与分析
5. 根因或可能原因
6. 风险等级
7. 已执行动作
8. 建议后续动作
9. 待确认事项

## 写作要求

- 结论先行。
- 区分事实、推断和假设。
- 不使用泛泛而谈的 AI 味表述。
- 面向技术主管时，突出风险、影响、优先级和资源需求。
